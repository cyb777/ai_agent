# 基于 LLM 智能体的自然语言数据分析与可视化平台

---

## 一、项目背景

### 项目前景分析

随着企业数字化转型加速，业务数据呈爆炸式增长，但传统数据分析高度依赖 SQL 编写能力和专业工具操作技能，导致数据利用率低下。本项目利用大语言模型（LLM）与智能体技术，构建以自然语言为交互入口的智能数据分析平台，用户无需掌握 SQL 语法，即可通过对话完成数据查询、图表生成、邮件通知等操作，显著降低数据分析门槛，提升业务决策效率。

---

## 二、项目概述

### 2.1 项目简介

本项目是一个基于 FastAPI + LangChain + Vue 的全栈智能体应用，采用阿里云通义千问 Qwen3-max 大模型，结合 MySQL 数据库、ECharts 可视化图表、QQ 邮箱 SMTP 服务，实现"自然语言输入 → 智能体分析 → 数据查询 → 图表生成"的一站式智能分析流程。

### 2.2 使用设备与运行环境

| 类别 | 配置 |
|------|------|
| 操作系统 | Windows 10/11 |
| 开发工具 | PyCharm（后端）、VS Code（前端） |
| Python 环境 | Conda 虚拟环境，Python 3.x |
| 数据库 | MySQL 8.0 |
| 前端运行 | Node.js |
| 大模型 | 阿里云通义千问 Qwen3-max |

### 2.3 实现功能

1. **自然语言转 SQL 查询**：用户用自然语言描述问题，智能体自动生成 SQL 并执行
2. **数据可视化**：查询结果自动转化为 ECharts 图表（柱状图、折线图、饼图等）
3. **智能数据分析**：自动生成数据表格 + 详细文本分析 + 可视化图表
4. **邮件验证码登录/注册**：邮箱验证码实现安全注册登录
5. **流式对话输出**：SSE（Server-Sent Events）实现实时流式响应

### 2.4 项目特点

- **低门槛交互**：全程自然语言对话，零 SQL 基础
- **模块化架构**：智能体、工具、模型三层解耦，易于扩展
- **结构化输出**：Pydantic 约束响应格式，前端精确解析
- **安全可控**：SQL 查询仅允许 SELECT，禁止增删改
- **流式体验**：边思考边输出，交互自然流畅

## 四、成员分工及职责描述

### 4.1 团队大体分工

团队采用 **"核心智能体 + 配套工具 + 基础架构"** 的垂直模块化分工模式，每人独立负责 1 个业务智能体及对应支撑组件，同时统一维护项目代码规范与接口标准，实现模块间低耦合、高内聚，保障整体系统可扩展与可维护性。

### 4.2 每位成员具体工作内容

#### 成员A：图表智能体 + 邮件通知工具

| 文件 | 工作内容 |
|------|----------|
| `app/ai/agent/echarts_agent.py` | 开发 EchartsAgent 图表智能体：实现数据库查询结果自动解析、转换为标准 ECharts JSON 图表数据格式 |
| `app/ai/tool/send_email_tool.py` | 封装邮件发送工具：集成 Logger 日志系统，实现邮箱配置通过环境变量注入，支持多场景邮件通知 |

#### 成员B：基础架构 + 数据分析智能体 + 大模型管理

| 文件 | 工作内容 |
|------|----------|
| `app/main.py` | 负责项目后端核心架构搭建：FastAPI 服务入口、lifespan 生命周期管理及全局 CORS 跨域配置 |
| `app/ai/agent/anlyze_agent.py` | 开发 AnlyzeAgent 数据分析智能体：实现"数据库查询→表格输出 + 文本分析→图表生成"三步式复合流程，支持结构化结果输出 |
| `app/ai/model/model.py` | 实现大模型单例管理：基于单例模式封装 MyModel 类，统一管理全局大模型实例，避免重复初始化与资源浪费 |

#### 成员C：登录智能体 + MySQL 数据工具

| 文件 | 工作内容 |
|------|----------|
| `app/ai/agent/system_agent.py` | 开发 SystemAgent 登录验证码智能体：实现 mysql_tool 与 send_email 双工具协作的流程化登录验证逻辑 |
| `app/ai/tool/mysql_tool.py` | 封装 MySQL 通用查询工具：完成 7 张业务表的文档注释（docstring），优化 SQL 执行异常处理与错误提示机制 |

| 文件 | 工作内容 |
|------|----------|
| `app/ai/agent/sql_question_agent.py` | 开发 SQLQuestionAgent 自然语言转 SQL 智能体：支持自然语言查询自动生成合法 SQL 语句 |

## 五、功能描述

### 5.1 项目实现功能列表

| 序号 | 功能 | 实现方式 | 技术栈 |
|------|------|----------|--------|
| 1 | 自然语言转 SQL 查询 | SQLQuestionAgent 接收自然语言问题，智能体调用 mysql_tool 自动生成并执行 SQL | LangChain Agent、PyMySQL、阿里云千问 |
| 2 | 数据可视化图表 | EchartsAgent 查询数据库后生成标准 ECharts JSON，前端渲染 | ECharts、LangChain Agent |
| 3 | 智能数据分析 | AnlyzeAgent 执行"查询→分析→图表"三步复合流程，返回结构化数据 | LangChain Agent、Pydantic |
| 4 | 邮箱验证码登录注册 | SystemAgent 双工具协作，查询 user_info 表验证邮箱后发送验证码 | LangChain Agent、QQ邮箱 SMTP、Redis |
| 5 | 流式对话输出 | SQLQuestionAgent 使用 astream 异步流式输出，SSE 协议推送到前端 | FastAPI StreamingResponse、AsyncIO |
| 6 | 对话记忆功能 | InMemorySaver 通过 thread_id 存储对话上下文，支持多轮连续对话 | LangGraph Checkpoint |
| 7 | 结构化响应 | Pydantic BaseModel 约束各智能体输出格式 | Pydantic、LangChain response_format |
| 8 | 跨域支持 | CORS 中间件配置允许前端跨域请求 | FastAPI CORSMiddleware |
